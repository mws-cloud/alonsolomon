## מה קורה אוטומטית

הקמת מסד הנתונים (טבלת `landing_content` + הרשאות לידים) יושבת כמיגרציה בתיקיית
`supabase/migrations/` (קובץ `20260717090000_landing_program_content_and_lead_policies.sql`)
ונמשכת אוטומטית בעת דחיפה — אין צורך להריץ אותה ידנית.
(`setup.sql` כאן הוא עותק זהה, כגיבוי להרצה ידנית ב-SQL Editor אם המיגרציה לא רצה.)

## מה נשאר ידני (דורש גישה ל-Dashboard)

### 1. קבלת גישה לפרויקט ה-Supabase

מי שמחזיק בפרויקט צריך להזמין אתכם:
**Supabase Dashboard ← Organization ← Team ← Invite member** (עם הרשאת Admin או Developer).

### 2. החלפת מפתח ה-anon השבור

⚠️ המפתח שמוטמע היום באתר החי אינו תקין (Supabase דוחה אותו). יש להעתיק את המפתח
האמיתי מ-**Settings ← API ← anon/public key** ולעדכן אותו ב-
`landing-app/src/lib/config.ts` (SUPABASE_ANON_KEY), ואז `npm run build` בתיקיית
landing-app ולדחוף מחדש. בלי זה הכניסה לפאנל וטעינת התוכן הנערך לא יעבדו.

### 3. יצירת משתמש כניסה ללקוח

1. בתפריט הצד: **Authentication ← Users ← Add user ← Create new user**
2. מזינים את האימייל של עו"ד סולומון וסיסמה ראשונית
3. מסמנים **Auto Confirm User** ✓
4. מוסרים ללקוח את האימייל והסיסמה — איתם הוא נכנס לפאנל בכתובת:
   `https://a-solomon.com/program/admin/`
